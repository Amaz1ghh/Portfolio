Stage Tertio Engineering

Développement WebData & IAPHPJavaScriptMySQLNASConduireAdministrerGérer

Configuration d'un NAS & Développement d'un POC (Proof of Concept) d'un site d'offres et de demandes de mobiliers selon des besoins métiers contenant :

Un système de connexion / inscription.

Une partie "Acheteur" permettant de faire des demandes de mobilier et de consulter les offres reçues. Les offres reçues possèdent un score selon l'adéquation avec la demande à laquelle elle répond.

Une partie "Vendeur" permettant de consulter les demandes reçues selon des préférences entrées, les offres envoyées et les offres ayant été refusées.

Une partie "Profil" permettant de changer ses informations personnelles ainsi que ses préferences, afin de personnaliser les demandes recues.

La recherche d'offres et de demandes peut être filtrée et triée. Les demandes peuvent également être "marquées en lu".

Parcours du Cavalier

Développement LogicielPythonPygameOptimiserCollaborer

Cet algorithme résoud le problème du parcous du cavalier :

Sur un plateau d'échec un cavalier doit parcourir tout le plateau sans repasser par la même case (parcours hamiltonien).

Le code source présente un parcours heuristique et non-heuristique. Chacun ayant la possiblité de trouver soit un soit tous les chemins.

Le parcours du cavalier est representé avec une interface graphique par un cavalier parcourant un (et un seul) chemin.

Génération Automatisée de Documentations

Développement WebDéveloppement LogicielPHPDockerBashAdministrerCollaborer

Ce projet permet de créer automatiquement des documentations utilisateurs à partir de fichiers C utilisant des commentraires spéciaux (à l'instar de doxygen) et des documentations techniques à partir de fichiers Markdown :

Un programme analyse les fichiers sources C à la recherche de commentaires spéciaux (par exemple, @brief) afin de créer une documentation technique. Cette documentation est configurée via un fichier de configuration et des options en ligne de commande.

Un autre programme convertit un fichier DOC_UTILISATEUR.md distinct en un manuel utilisateur.

Un script bash tente d'automiser ce processus dans Docker, en gérant la copie et l'exécution des fichiers.

Le style est géré par CSS dans le répertoire themes, et la structure HTML provient de modèles situés dans le répertoire data.
